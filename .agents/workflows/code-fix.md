# `/code-fix` — Corregir Malas Prácticas

Use this workflow AFTER running `/code-audit`. It reads the findings report and applies targeted fixes guided by project skills.

## 1. Load Report

1. Read the artifact `audit_report.md` generated by `/code-audit`.
2. If it does not exist, **stop** and instruct the user to run `/code-audit` first.
3. Parse the findings list. Do NOT load any source files yet.

## 2. Skill Selection

Before fixing any finding, load the skill that governs that category.
**Read the SKILL.md file with `view_file` before writing any code.**

| Finding Category | Skill to Load |
|---|---|
| Rust errors, ownership, `.unwrap()`, `clone()` | `rust-best-practices` |
| Rust async, blocking calls, Tokio patterns | `rust-async-patterns` |
| Tauri commands, IPC, capabilities/permissions | `tauri-v2` |
| Tauri state sync, events, Zustand, React Query | `tauri-state-sync` |
| React components, CSS, UI patterns | `frontend-design` |
| UI accessibility and design guidelines | `web-design-guidelines` |
| Masonry, virtualization, thumbnail performance | `virtuoso-masonry-performance` |
| yt-dlp process management, parsing | `yt-dlp-integration` |
| FFmpeg sidecar, thumbnails, conversion | `ffmpeg-media-processing` |

## 3. Triage

1. Present the user with the full findings list, grouped by severity (HIGH → MEDIUM → LOW).
2. Ask: *"Which findings should I fix? (all / HIGH only / list specific IDs)"*
3. Wait for the user's response before proceeding. Do not auto-fix.

## 4. Fix Loop

For each approved finding, follow this strict sequence:

```
FOR each approved finding:
  1. Read the relevant SKILL.md (if not already loaded for this category).
  2. Read ONLY the specific file + surrounding context (±20 lines) of the finding.
  3. Apply the minimal fix that resolves the issue per skill guidelines.
  4. Do NOT refactor unrelated code in the same file.
  5. Verify: does the fix break any direct callers visible in the file? If yes, fix those too.
  6. Update the audit_report.md: mark the finding as [FIXED].
  7. Move to the next finding.
```

## 5. Completion

1. Update `audit_report.md` marking all resolved findings as `[FIXED]`.
2. `notify_user` with a summary: how many findings fixed, how many skipped, and any that could NOT be fixed automatically (require manual review).
3. Recommend running `cargo clippy` (Rust) or `npm run lint` (frontend) to confirm no regressions.
